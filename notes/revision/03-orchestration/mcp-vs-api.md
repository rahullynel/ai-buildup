# MCP vs API

## What is MCP
- Model Context Protocol: standardized interface for model/tool integrations.
- Helps tools expose capabilities consistently to model-driven systems.

## API-first approach
- Best when integrations are few and stable.
- Lower abstraction overhead.
- Easier direct performance tuning per endpoint.

## MCP-first approach
- Best when tool ecosystem is broad/changing.
- Standardized integration pattern across tools.
- Better portability in multi-tool agent architectures.

## Quick decision rule
- Small, fixed app: API-first is fine.
- Growing multi-tool platform: MCP often pays off.
