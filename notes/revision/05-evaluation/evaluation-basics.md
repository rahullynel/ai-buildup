# Evaluation Basics

## Why evaluation matters
- Demos can look good while production fails silently.
- Need measurable quality gates and regression checks.

## Core quality dimensions
- Faithfulness: answer supported by evidence.
- Relevance: answer addresses the question.
- Context precision: retrieved context is mostly useful.

## Retrieval checks
- Recall@k
- Precision@k
- MRR / nDCG

## Good evaluation workflow
- Evaluate retrieval quality separately.
- Evaluate generation quality separately.
- Then evaluate end-to-end behavior.
